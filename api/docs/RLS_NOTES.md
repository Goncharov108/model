# RLS: заметки до первой миграции

Краткий конспект для будущего Postgres, без исполняемого SQL (см. `migrations/README.md`).

## Принципы

- **Deny by default**: нет политики — нет строк для роли приложения.
- Сервисные учётные записи, обходящие RLS, держать вне горячего пути чтения пользовательских данных.
- Политики выражать через **текущего субъекта** сессии (`SET LOCAL` / `current_setting`), а не через «магические» ID в JWT без проверки.

## Связь с кодом фронта

Типы в `web/src/domain/canonDomainModel.ts` — ориентир для будущих таблиц `subjects`, `layers`, `relations`; миграции должны оставаться источником правды по схеме, UI — потребитель.
