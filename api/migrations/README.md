# Миграции PostgreSQL (заготовка)

Каталог зарезервирован под **версионируемые SQL-миграции** после появления реального API-слоя (см. фазу 4 в `PLAN_REALIZATSII.md`).

## Ориентир из канона-3

- Транзакционный реестр в PostgreSQL с **RLS** как последней линией при ошибках в API.
- Политики на уровне строк согласованы с графом отношений (OpenFGA / доменная модель в `web/src/domain/canonDomainModel.ts`).

## Следующие шаги (когда появится `api` на Nest или ином стеке)

1. Выбрать инструмент (`node-pg-migrate`, `goose`, `atlas`, SQL-файлы + собственный runner).
2. Первая миграция: расширения, схема `app`, таблица-заглушка и **включение RLS** с `FORCE ROW LEVEL SECURITY` только после появления сервис-ролей.
3. Отдельный документ с матрицей «роль БД ↔ политика RLS» — не дублировать сюда до первой миграции.

Пока файлов `.sql` нет — это намеренно.
