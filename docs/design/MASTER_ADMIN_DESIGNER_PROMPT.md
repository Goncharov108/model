## Архитектура навигации

Схема (отдельно, не вложенный блок кода): главное меню fixed → при MA оверлей (Обзор, Окружение, Пользователи, Продвинутые) → внутри Продвинутых колонка Поток/Структуры/План/Оркестр/Домен/Справка. Полная mermaid — в `docs/design/MASTER_ADMIN_DESIGNER_PROMPT.md` вне блока «Скопировать».

**Главное меню** — `fixed`, на всю высоту viewport (`w-60`), контент с отступом `pl-60`.

При `/master-admin/*` панель Мастер-админ **наслаивается** поверх главного меню (amber, blur, тень, `z-50`), не сдвигает layout. Кнопка **«← Общее меню»** → `/`.

**Продвинутые настройки:** подменю **не** в панели Мастер-админ — отдельная колонка/чипы в рабочей области (`AdvancedSettingsNav` + шапка в `AdvancedSettingsShell`). Подсказка в панели MA: «Разделы — в колонке справа».

## Мастер-админ — верхний уровень

### 1. Обзор `/master-admin/overview`

- PageHeader: «Мастер-админ» / «Обзор»
- Текст: инструменты во «Продвинутых настройках»
- Кнопка → Поток (advanced)

### 2. Окружение `/master-admin/environment` ★ инфраструктура

**Назначение:** документация стека и деплоя (без секретов в полях).

**Карточки (SurfaceCard):**

1. **Репозиторий и CI** — поля: URL GitHub, имя repo, visibility (select), ветка, workflow, статус CI, заметки  
2. **Домен (reg.ru)** — домен, DNS-записи (textarea), SSL статус, дата проверки, заметки  
3. **Сервер** — хост, ОС, SSH user, Node, пути web/api, заметки  
4. **Стек приложения** — три таблицы (редактируемые строки):  
   - На сервере / в проде  
   - В репозитории  
   - Запланировано (канон-3)  
   Колонки: Слой | Технология | Версия | Где | Заметка | удалить  
   + поле «Ссылка на канон-3»  
5. **Секреты и границы** — что не в Git; куда на сервере; заметки  
6. **Экспорт и импорт** — кнопки: Экспорт JSON, Скопировать чеклист деплоя, Сброс к шаблону; textarea импорта; подсказка про `docs/environment/`

**Persist:** `model-environment-docs-v1`  
**Акцент полей:** amber focus ring (отличие от violet в аккаунте)

### 3. Пользователи `/master-admin/users`

- Статистика: чипы «Всего» + по ролям (Владелец, Разработчик, Наблюдатель, Гость)
- Карточка «Поиск и фильтры»: строка поиска, фильтр роли, сортировка, «Добавить»
- Карточка редактора (создание/правка): имя, телефон, роль
- Таблица пользователей: аватар, имя, телефон, роль, дата, действия
- ConfirmDialog на удаление

**Persist:** `model-users-v1`

### 4. Продвинутые настройки `/master-admin/advanced/*`

**Layout:** шапка раздела (violet eyebrow, заголовок = текущая вкладка, hint из `ADVANCED_SETTINGS_NAV`) + **внутренняя навигация** (desktop: колонка `w-56` с label+hint; mobile: горизонтальные чипы) + контент workspace. Дублирующие `PageHeader` на экранах **убраны**.

| Маршрут | Вкладка | Ключевые блоки | Статус |
|---------|---------|----------------|--------|
| `/master-admin/advanced` | Поток | Текст, локальный/внешний анализ, каноны, экспорт, empty states | готово |
| `.../structures` | Структуры | Реестр, поиск, заметки, «Открыть на Потоке» | готово |
| `.../plan` | План | Опросник, JSON | готово |
| `.../orchestra` | Оркестр | Роли, задачи, дирижёр | готово |
| `.../domain` | Домен | Слои/субъекты канона-3 | готово |
| `.../help` | Справка | Онбординг, UI_OVERVIEW | готово |

**Persist:** model-stream-v1, model-analysis-v1, model-planning-v1, model-orchestra-v1, model-domain-v1, model-structure-notes-v1

## Задачи дизайнера (deliverables)

1. **Карта экранов** (все маршруты выше) с пометкой «готово в коде» / «заглушка».  
2. **Макет панели Мастер-админ** — оверлей поверх main, кнопка «← Общее меню», 4 пункта MA_NAV.  
3. **Макет «Продвинутые настройки»** — шапка + внутренняя колонка навигации + контент Потока (эталон).  
4. **Макет «Окружение»** — 6 карточек, таблицы стека, amber focus.  
5. **Макет «Пользователи»** — статистика, фильтры, таблица, mobile.  
6. **Токены** — цвета, отступы, типографика (согласовать amber/violet).  
7. **Не рисовать** поля для паролей/токенов с реальными значениями — только placeholders.

## Примечания для владельца

- После правок DNS/SSL обновляйте скриншоты и поле «Статус SSL» на макете «Окружение».
- Для Canvas в Cursor можно дублировать таблицы стека из вкладки «Окружение» как интерактивный артефакт.
- Чеклисты в брифе разделены: **код** (готово) vs **макеты дизайнера** (отдельная работа).
