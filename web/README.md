# Веб-интерфейс (локально)

Репозиторий и проект в Git называются `model` (корневой каталог). Каталог `web/` — локальный фронтенд этой кодовой базы. Название «Мастер» в канонических PDF и мастер-тексте в `private/` относится к продуктовой идее из документов, не к имени репозитория.

Канонические решения и контекст: корень репозитория, `SWOD_ZAKONOV.txt` (законы 6 и 12 — веб-интерфейс и режим «Оркестр»). Дорожная карта фаз и чеклист: `../PLAN_REALIZATSII.md` (относительно каталога `web/`, в корне репозитория `model`).

## Команды

```bash
cd web
npm install
npm run dev
```

Открой в браузере адрес из вывода Vite (обычно `http://localhost:5173/`).

```bash
npm run build
npm run preview
npm run test
npm run test:watch
```

После push в GitHub workflow **CI** (`.github/workflows/ci.yml`): lint, test, build в `web/`, смок `api/server.mjs` (`/health`).

## Node

В корне репозитория есть `.nvmrc` (сейчас `22`). Убедись, что в `PATH` используется соответствующий Node (например через nvm).
