## Три опоры (всё важно)

Vault задуман под **три равноправные линии** — без «главной» и «второстепенной»:

| Опора | Папки | Зачем |
|--------|--------|--------|
| **Личные заметки** | `journal/`, отражения в `canon/` | Поток мыслей, дневник, личный разбор канона; сакральное и интимное не дублировать из `private/` — там закон. |
| **Документация по model** | `model/`, `infrastructure/` | Идеи продукта, дорожная карта, черновики к `docs/`, связь с Мастер-админом и репозиторием. Факты без секретов — дубли в `docs/environment/` и коде. |
| **Промпты под агентов** | `prompts/` | Cursor, Telegram, n8n — версии, дата в имени файла, краткий контекст «кому и зачем». |

Остальное — **связующий слой**:

- **`canon/`** — опоры из `CANON_*.txt`, ваши ссылки и выводы (не сырой мастер-текст).
- **`infrastructure/`** — мост между головой и `docs/environment/` (сервер, DNS, деплой).

Главная точка входа: **`00-индекс.md`**.

**Структура репозитория:** [[project-structure]] (генерируется скриптом).

**Граф Obsidian:** линии появляются только из **`[[wikilinks]]`**. В каждом разделе есть заметка-хаб с уникальным именем (не несколько `README`). Свяжите свои заметки с хабом: внизу файла строка `[[Журнал]]` или `[[Промпты]]`.
