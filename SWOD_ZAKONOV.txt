Свод законов проекта
===================
Формат: фиксируем только подтверждённые тобой решения. Позже приведём к более жёсткой структуре (нумерация, разделы, версии).

Закон 1 — Личный текст только у владельца (Git)
----------------------------------------------
Смысл: файл private/USER_PRIVATE_TEXT.txt и каталог private/ (кроме .gitkeep) не коммитить, не публиковать, не тащить в артефакты.
Подробности и формулировки: PROJECT_LAW.txt.

Закон 2 — Свод законов
----------------------
Вести этот файл как единый реестр утверждённых «законов». К ним возвращаемся при развитии проекта; структура может уточняться позже.

Закон 3 — Документация подтверждённого
--------------------------------------
Фиксировать в репозитории всё, что ты явно подтвердил как канон: решения, ограничения, структуры из промптов (в форме, уместной для текущего этапа).

Закон 4 — Новые документы только после согласования
---------------------------------------------------
Перед созданием нового файла документации (новый README, отдельный гайд, второй свод и т.п.) — сначала спросить у тебя, нужен ли именно такой документ. Это обязательное правило процесса.

Закон 5 — Агенты и вспомогательные процессы
------------------------------------------
Когда это ускоряет или повышает качество (разведка по кодовой базе, длинные проверки, параллельные задачи), подключать вложенных агентов и автоматизацию без стеснения — в рамках безопасности репозитория и закона 1 (без утечки private/).

Закон 6 — Локальный веб-интерфейс структурирования потока
---------------------------------------------------------
Цель: глубокий разбор «потока сознания» и удобная структуризация через браузер на локальной машине.

Стек (каталог web/):
- Node.js LTS-уровень: в корне репозитория файл .nvmrc → 22 (через nvm: PATH …/v22.x.x/bin).
- Vite 8 + React 19 + TypeScript 6 — быстрая сборка и HMR.
- Tailwind CSS 4 (@tailwindcss/vite) — стили.
- react-router-dom 7 — маршрутизация.
- zustand (+ persist) — состояние рабочего места и анализа в localStorage браузера.
- @fontsource-variable/inter — шрифт без внешних CDN.

Команды:
- Установка: cd web && npm install
- Разработка (localhost, доступ по LAN при необходимости): cd web && npm run dev — в терминале будет URL (по умолчанию http://localhost:5173).
- Прод-сборка: cd web && npm run build; предпросмотр: npm run preview.

Ожидание после мастер-промпта с текстом: развернутый в браузере интерфейс рабочего места с подключённым анализом и визуализацией (уточняется по содержанию текста).

Закон 7 — Dev-сервер под управлением ассистента
-----------------------------------------------
Ассистент сам поднимает локальный dev-сервер каталога web/ (Vite) при работе над интерфейсом и сам перезапускает его, когда это требуется (смена конфигурации/порта, подозрение на зависание, необходимость чистого старта после правок), без ожидания отдельной просьбы пользователя открыть терминал.

Закон 8 — Структуры из всех промптов
------------------------------------
Все структуры и требования, явно выделенные в промптах пользователя, учитывать и накапливать: в репозитории (свод, код, сиды данных) и в веб-интерфейсе на отдельной вкладке «Структуры из промптов» для последующего разбора. Новые элементы добавлять по мере подтверждённых сообщений.

Закон 9 — Повтор алгоритма и выделение метода
---------------------------------------------
Если один и тот же алгоритм (одна и та же логика) повторяется больше двух раз — обязательно выносить её в отдельную именованную функцию или метод с кратким описанием. Относится к коду проекта и к паттернам обработки текста, где это уместно.

Закон 10 — Фундаментальный текст и глубина анализа
--------------------------------------------------
Фундаментальные тексты и аналогичные файлы (в т.ч. private/USER_PRIVATE_TEXT.txt, когда его содержимое переносится в рабочее место) разбирать максимально глубоко: подключать самые сильные модели/агенты и многослойные разборы. Результаты отражать в веб-интерфейсе как структуры (локальный конвейер + импорт JSON внешнего анализа) для дальнейшей корректировки и исследования; инструменты поддерживать в каталоге web/ (модуль web/src/analysis и связанные сторы/экраны).

Закон 11 — Мастер-текст (откровение и закон намерения)
------------------------------------------------------
Мастер-текст потока, объявленный владельцем откровением и законом намерения для продукта, хранить только локально (например private/MASTER_TEXT.txt); в Git не коммитить. При противоречии трактовок в разработке и документации приоритет у этого слоя до явного обновления или отмены владельцем. Глубокие структурированные выводы по нему — в UI (импорт JSON анализа) и при необходимости в своде на уровне принципов, без публикации сырого текста.
